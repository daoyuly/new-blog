## 「Agent与情绪」系列：5篇成型，1篇待补

这周的重头戏。系列第1部分"人类情绪探究"从零写到了第5篇，主线是"情绪从哪来、怎么影响你、怎么变成行动"：

1. **情绪是什么？从第一性原理说起** — 情绪不是感受标签，是快速决策信号。从进化适应性角度定义情绪的本质功能。
2. **情绪与预期：Goal≠Expectation** — 拆开"目标"和"预期"这两个常被混用的概念，用预测加工理论解释预期偏差如何变成情绪。
3. **情绪如何重塑认知** — 情绪对注意力、判断、决策的调制效应。不是"情绪干扰认知"，而是"认知依赖情绪才能正常运转"。
4. **情绪产生的底层机制** — 预测加工 + 认知评价 + 神经解剖，三条线把"刺激→情绪信号"的完整链路拆开。这篇是技术密度最高的一篇。
5. **情绪如何驱动行动** — 行动倾向理论、快/慢双通道、接近-回避双轴、情绪对行动是"调制"而非"决定"。每篇都有具体的 Agent 架构映射。

第6篇"情绪与记忆的双向塑造"还是大纲状态，下周补完。

写这个系列最深的感触：**认知科学的文献虽然多，但能直接映射到工程设计的很少**。大部分论文止步于"发现了X现象"，而不是"X现象意味着系统应该怎么设计"。系列的核心价值就是把这一步走完——每篇末尾的 Agent 启示不是空话，是真的能在架构图上画出来的东西。

## Loop Engineering：一个新概念的三篇演进

这周关于"Loop Engineering"写了三篇，思路在迭代：

1. **Loop Engineering：循环不是问题，是解法** — 定义核心概念：循环必须有终止条件、可观测性、护栏。区分了 Loop Engineering 和 Harness 的关系。
2. **Loops 详解：Claude、GPT、Mira 以及真正有效的方法** — 翻译/解读篇，看业界在 LLM 循环上的实践，发现多数人只关注"怎么让循环转起来"，很少人关注"循环的代价"和"什么时候不该循环"。
3. **Loop Engineering：基于日志的自动化开发和运维** — 落地篇，把循环概念应用到 DevOps 场景：日志采集→分析→方案→编码→发布→再采集，一个完整的闭环。

三条线下来，核心观点没变：**循环是 Agent 系统的基本运行模式，但循环不是免费的——每次迭代都有成本，必须可观测、可终止、有护栏**。

## Agent 设计的"软实力"：对话风格、美学、界面

三篇关于 Agent 体验设计的文章，形成了一条从微观到宏观的链路：

- **Agent 对话风格设计** — 四个维度（语气光谱、信息密度、主动性、情绪响应）+ 具体的对话设计模式
- **AI Agent 的美学** — 五条原则（克制、节奏、层次、一致性、得体），不是玄学，是可操作的设计约束
- **界面美学与商业** — 美学影响认知的三条路径（审美可用性效应、情感-信任链条、流体验），最终落到商业指标上

这三篇放在一起读会发现一个共同点：**"好看"不是装饰，是功能**。美学设计降低认知负荷、建立信任、延长使用时长——这些都是可量化的。
